# Custom Pipelines Dataset — Creation Methodology

## Overview

This document describes the complete methodology used to create the **Custom Pipelines** dataset — a collection of 100 end-to-end ML data pipelines spanning 10 domains. The dataset was constructed through a multi-stage process combining real-world ML projects with LLM-generated pipeline trajectories.

---

## 1. Dataset Architecture

The 100 pipelines are organized into two categories based on their origin:

### Category A — Hand-Curated Domain Pipelines (001–013)

**Source:** Standard machine learning projects from GitHub and domain-specific data repositories.

These 13 pipelines were curated from real-world ML project templates covering finance, healthcare, HR, energy, e-commerce, supply chain, insurance, and manufacturing. Each was selected to represent a complete, production-quality ML workflow with:

- **Realistic data schemas** (e.g., FICO scores, medical records, HR databases, stock market data)
- **Multi-source integration** (CSV + JSON + Excel, 2–4 input files)
- **Advanced ML techniques** (ensemble models, SHAP explainability, fairness audits, cohort analysis)
- **Business context** and regulatory compliance considerations

## 2. Source Dataset: DataScience-Instruct-500K

### Overview

The primary source for Category B pipelines is the **[DataScience-Instruct-500K](https://huggingface.co/RUC-DataLab/DataScience-Instruct-500K)** dataset — a large-scale instruction dataset for training data science agents, released by Renmin University of China's DataLab.

- **Size:** 500,000 records
- **Format:** Multi-turn agentic trajectories with Analyze → Code → Execute → Answer blocks
- **Source Model:** Generated by DeepAnalyze-8B, an agentic LLM trained for autonomous data science
- **Paper:** [DeepAnalyze: Agentic Large Language Models for Autonomous Data Science](https://arxiv.org/abs/2510.16872) (Zhang et al., 2025)

### Dataset Structure

Each record in DataScience-Instruct-500K contains:

```json
{
  "id": "case_identifier",
  "messages": [
    {"role": "user", "content": "Data science instruction with file descriptions"},
    {"role": "assistant", "content": "<Analyze>...</Analyze><Code>...</Code><Execute>...</Execute><Answer>...</Answer>"}
  ],
  "evaluation": {
    "ability": ["data_cleaning", "visualization", "statistical_analysis"],
    "difficulty": 4,
    "quality": 4.5
  }
}
```

The assistant response contains **multi-step agentic trajectories** where the model:
1. **Analyzes** the data and plans the next step
2. **Writes Code** to execute the planned analysis
3. **Executes** the code and observes outputs
4. **Answers** with a final research report

### Download Process

```bash
# Download the full dataset from HuggingFace (1.30 GB)
python download_full_dataset.py

# Split into individual JSON files by difficulty
python split_dataset.py
```

The download script (`download_full_dataset.py`) uses the HuggingFace `datasets` library:
```python
from datasets import load_dataset
ds = load_dataset("RUC-DataLab/DataScience-Instruct-500K", split="train")
ds.to_parquet("dataset_output/full_dataset.parquet")
```

## 3. Pipeline Generation Pipeline

### Architecture

The pipeline generation system transforms raw LLM trajectories into complete, runnable pipeline artifacts through a multi-stage process:

```
DataScience-Instruct-500K
        │
        ▼
┌──────────────────────────┐
│ 1. Record Parsing         │  Extract Analyze/Code/Execute blocks
│    (parse_record)          │  Identify successful execution steps
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────┐
│ 2. Synthetic Data Gen     │  Parse table schemas from Execute output
│    (generate_synthetic)    │  Create CSV/JSON/XLSX with sample data
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────┐
│ 3. Script Generation      │  Clean code blocks into standalone scripts
│    (clean_script)          │  Add argparse, fix file references
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────┐
│ 4. Documentation          │  Generate pipeline.md with business context
│    (generate_pipeline_md)  │  Extract step descriptions from Analyze blocks
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────┐
│ 5. Orchestration          │  Generate Argo Workflow pipeline.yaml
│    (generate_argo_workflow)│  Create DAG of step templates
└──────────────────────────┘
```

### Key Components

#### 3.1 Record Parser (`generate_pipeline.py`)

The `parse_record()` function extracts structured information from each dataset record:

- **Step identification:** Finds `<Analyze>`, `<Code>`, `<Execute>` blocks in order
- **Error detection:** Marks steps where Execute starts with `[Error]`
- **File I/O extraction:** Parses `pd.read_csv()`, `.to_csv()`, etc. to identify data dependencies
- **Dependency resolution:** Determines initial inputs (files read but never produced)

#### 3.2 Synthetic Data Generator

The `generate_synthetic_data()` function creates realistic input data files by parsing table schemas from the Execute output blocks:

- **Schema parsing:** Extracts column names and dtypes from Execute output tables
- **Sample data:** Uses the sample rows shown in Execute output where available
- **Format handling:** Generates CSV, JSON, or XLSX based on file extension references in code

#### 3.3 Script Cleaner

The `clean_script()` function transforms raw code blocks into standalone, runnable Python scripts:

- **Import collection:** Gathers all imports at the top
- **argparse integration:** Replaces hardcoded file paths with command-line arguments
- **Variable substitution:** Maps `--input_csv` arguments to variable names used in code
- **Step documentation:** Adds docstrings extracted from Analyze block descriptions

#### 3.4 Documentation Generator

The `generate_pipeline_md()` function creates comprehensive pipeline documentation:

- **Metadata extraction:** Pulls difficulty, quality, and ability tags from evaluation
- **Business context:** Generates domain-specific business implications
- **Architecture diagram:** Creates ASCII art pipeline flow diagrams
- **Step details:** Lists each step with inputs, outputs, script references, and purpose

#### 3.5 Argo Workflow Generator

The `generate_argo_workflow()` function creates Kubernetes-native workflow definitions:

- **DAG structure:** Each step becomes a sequential template
- **Volume mounts:** Input data from `/data`, intermediate outputs shared via `/mnt/output`
- **Resource allocation:** 512Mi–1Gi memory, 500m–1000m CPU per step

### Batch Processing

```bash
# Generate N pipelines from the dataset
python batch_generate.py 100
```

The batch generator (`batch_generate.py`) processes records sequentially, handling errors gracefully and reporting success/failure rates.

## 4. DeepAnalyze-8B: The Generation Model

### Paper Reference

> **DeepAnalyze: Agentic Large Language Models for Autonomous Data Science**
> Shaolei Zhang, Ju Fan, Meihao Fan, Guoliang Li, Xiaoyong Du
> Renmin University of China
> arXiv:2510.16872, 2025

### Key Contributions

1. **Agentic Training for Data Science:** DeepAnalyze-8B is trained to autonomously perform end-to-end data science tasks — from raw data ingestion to research report generation — in real-world environments.

2. **Multi-Step Trajectories:** The model generates Analyze → Code → Execute → Answer sequences, mimicking the iterative workflow of a human data scientist exploring and analyzing data.

3. **DataScience-Instruct-500K:** The training dataset containing 500K instruction-trajectory pairs across diverse data science tasks with varying difficulty levels and required abilities.

4. **Ability Coverage:** The model exercises 12 core data science abilities:
   - Data loading, cleaning, standardization
   - Exploratory data analysis
   - Statistical analysis and hypothesis testing
   - Data visualization
   - Feature engineering
   - Model training (classification, regression, clustering)
   - Model evaluation
   - Data integration and merging
   - Data validation and quality checks
   - Error recovery and robust loading
   - Geospatial analysis
   - Final synthesis and reporting

### Model Architecture

DeepAnalyze-8B is based on Qwen2.5-7B-Instruct, fine-tuned with agentic training on the DataScience-Instruct-500K dataset. The training enables the model to:

- Understand raw data files and their schemas
- Plan multi-step analytical workflows
- Write and execute Python code in real environments
- Recover from errors and adapt the analysis plan
- Generate comprehensive research reports

## 5. Post-Generation Refinement

After initial generation, all 100 pipelines underwent extensive refinement:

### Data Refinement
- **Category A:** Realistic data schemas with proper statistical distributions, multi-source formats (CSV, JSON, Excel), and appropriate sample sizes (50–21,600 records)
- **Category B:** Domain-specific synthetic data (sports: 400–520 records with player stats; transportation: 500–800 records with transit data; education: 300 student records)

### Script Standardization
- All scripts converted to a consistent 6-step architecture: Load → Clean → Features → EDA → Model → Report
- Category A pipelines retain additional domain-specific steps (10–16 total)
- Added `--data_dir` and `--output_dir` arguments for portability
- Hardened JSON serialization with `default=str` for numpy compatibility
- Syntax validation across all 624 scripts

### Documentation Accuracy
- All 100 `pipeline.md` files verified against actual scripts and data files
- Step counts synchronized between documentation and implementation
- Business context aligned with actual domain and techniques

### Container Readiness
- Added per-pipeline `Dockerfile` (python:3.12-slim base)
- Auto-detected `requirements.txt` from actual import statements (5–10 packages)
- Rewrote `pipeline.yaml` with DRY `run-step` template and custom image references
- Output directory shared via `emptyDir` volume between steps
